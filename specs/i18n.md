## 1. 架构概览

i18n 系统分为三层，数据单向流动：

```
┌─────────────────────────────────────────────────────────────┐
│  JSON 源文件（Dart: slang 格式）                              │
│  apps/desktop/lib/src/i18n/{en,zh-Hans}.i18n.json          │
└────────────┬───────────────────────────────────────┬────────┘
             │                                       │
    ┌────────▼────────┐                  ┌───────────▼──────────┐
    │ Dart 生成代码    │                  │ macOS 生成文件        │
    │ (slang)         │                  │ (codegen.py →        │
    │ strings.g.dart  │                  │  macos_i18n.py)      │
    │ strings_en.g.dart│                  │                      │
    │ strings_zh_Hans  │                  │ *.lproj/Localizable  │
    │   .g.dart       │                  │   .strings           │
    └────────┬────────┘                  │ LocaleKeys.swift      │
             │                           └───────────┬──────────┘
             │                                       │
    ┌────────▼────────┐                  ┌───────────▼──────────┐
    │ Flutter Widget  │                  │ SwiftUI Settings     │
    │ (Dart)          │                  │ (macOS Native)       │
    │ t.common.ui     │                  │ LocaleKeys.common    │
    │   .button.ok   │                  │   .ui.button.ok     │
    │ t.app.tray      │                  │ LocaleKeys.app       │
    │   .context_menu │                  │   .tray.contextMenu │
    └─────────────────┘                  └──────────────────────┘
```

另外，**语言名称列表**（支持哪些语言作为翻译目标）来自独立的 JSON 数据文件，会自动同步到 Rust 和 i18n JSON 中：

```
scripts/generate/data/languages.json
  └─→ crates/engine/src/languages.rs      （Rust 侧语言列表）
  └─→ *.i18n.json → common.language.*     （UI 显示各语言名称）
```

### 关键约束

| 约束 | 说明 |
|------|------|
| **JSON 是唯一源** | 所有翻译文本以 `*.i18n.json` 为准，不要直接编辑生成文件 |
| **单向生成** | 生成的文件（`.g.dart`、`.strings`、`LocaleKeys.swift`）不应手工修改 |
| **macOS 与 Dart 共享同一套 key** | 保证两个 UI 层显示一致；key 层级与 JSON 嵌套结构一一对应 |
| **参数占位符** | 使用 `{}` 作为占位符，运行时替换（Dart/Swift 均支持） |

## 2. Flutter 侧（Dart）

### 2.1 技术选型

- **[slang](https://pub.dev/packages/slang)** v4.13 — 代码生成式 i18n，编译时类型安全
- **[slang_flutter](https://pub.dev/packages/slang_flutter)** — Flutter 集成
- **[slang_build_runner](https://pub.dev/packages/slang_build_runner)** — Build_runner 生成器
- **flutter_localizations** — Flutter 内置 Material 本地化

### 2.2 源文件格式

JSON 嵌套结构，key 名使用 `snake_case`：

```json
{
  "common": {
    "ui": {
      "button": {
        "ok": "OK",
        "cancel": "Cancel"
      }
    },
    "language": {
      "en": "English",
      "zh_cn": "Chinese (Simplified)"
    },
    "provider": {
      "google": "Google",
      "deepl": "DeepL"
    },
    "theme_mode": {
      "dark": "Dark",
      "light": "Light"
    },
    "translation_mode": {
      "auto": "Auto",
      "manual": "Manual"
    },
    "word_pronunciation": {
      "us": "US",
      "uk": "UK"
    }
  },
  "app": {
    "tray": {
      "context_menu": {
        "quit": "Quit"
      }
    }
  },
  "settings": {
    "general": {
      "title": "General"
    }
  }
}
```

文件名规则：`{locale}.i18n.json`

| 文件 | 区域标识 |
|------|---------|
| `en.i18n.json` | English |
| `zh-Hans.i18n.json` | 简体中文 |

### 2.3 使用方式

```dart
import 'package:flutter/widgets.dart';
import 'package:flutter_localizations/flutter_localizations.dart';
import 'src/i18n/i18n.dart';

// 在 Widget 中使用
Text(t.common.ui.button.ok);
Text(t.settings.general.title);

// 带参数
Text(t.settings.providers.deleteDialog.message("Google Translate"));

// 设置语言
await context.setLocale(Locale('zh', 'Hans'));

// 获取当前语言
final locale = context.locale;

// 提供 localization delegates
MaterialApp(
  localizationsDelegates: context.localizationDelegates,
  supportedLocales: context.supportedLocales,
);
```

### 2.4 生成的 Dart 文件

| 文件 | 说明 |
|------|------|
| `strings.g.dart` | 总入口，定义 `AppLocale` 枚举、`Translations` 基类 |
| `strings_en.g.dart` | 英文翻译实现（part of `strings.g.dart`） |
| `strings_zh_Hans.g.dart` | 中文翻译实现（延迟加载，`deferred as l_zh_Hans`） |
| `i18n.dart` | 扩展方法（`context.setLocale`、`context.locale` 等） |

### 2.5 翻译函数（`formatTranslation`）

`i18n.dart` 中定义了一个工具函数用于替换占位符：

```dart
String formatTranslation(String value, {List<String> args = const []}) {
  var result = value;
  for (final arg in args) {
    result = result.replaceFirst('{}', arg);
  }
  return result;
}
```

## 3. macOS 侧（Swift）

### 3.1 文件布局

```
apps/desktop/macos/Runner/
├── en.lproj/
│   ├── InfoPlist.strings      # （手动维护）
│   └── Localizable.strings    # 自动生成
├── zh-Hans.lproj/
│   └── Localizable.strings    # 自动生成
└── Shared/
    └── I18n/
        ├── AppLocale.swift    # 手动维护，语言切换逻辑
        └── LocaleKeys.swift   # 自动生成，类型安全 key 常量
```

### 3.2 语言切换（`AppLocale`）

```swift
final class AppLocale: ObservableObject {
  static let shared = AppLocale()

  // 语言变更时自动通知 SwiftUI 刷新
  @Published private(set) var languageCode: String

  func setLocale(_ languageCode: String)
  func localizedString(_ key: String) -> String
}
```

- 语言偏好存储在 `UserDefaults.standard` 的 `macSettings.appearance.language` key 下
- 设置语言时同时写入 `AppleLanguages` 以影响系统行为
- 区域归一化逻辑：`en` → `en`，`zh`/`zh-Hans`/`zh-Hant` → `zh-Hans`（目前仅支持中英双语）

### 3.3 使用方式

```swift
import Shared/I18n/LocaleKeys

Text(LocaleKeys.common.app.name.tr())
// 或带参数
Text(LocaleKeys.settings.providers.deleteDialog.message.tr("Google Translate"))
```

### 3.4 代码生成

`scripts/generate/macos_i18n.py` 负责从 `*.i18n.json` 生成：

1. 每个 `{locale}.i18n.json` → `{locale}.lproj/Localizable.strings`
2. 所有 key 集合 → `Shared/I18n/LocaleKeys.swift`

生成的 Swift key 常量用嵌套 `enum` 描述层级结构，leaf 节点为 `LocaleKey`：

```swift
enum LocaleKeys {
  enum common {
    enum app {
      static let name = LocaleKey("common.app.name")
    }
  }
}
```

> **注意：** Swift 关键字冲突时自动用反引号包裹（如 `continue` → `` `continue` ``）

## 4. 语言数据管线

### 4.1 数据源

`scripts/generate/data/languages.json` 是**所有支持的语言列表**（约 170+ 语言），每条记录包含：

```json
{
  "en": {
    "local_name": "English",
    "translations": {
      "en": "English",
      "zh-Hans": "英语"
    }
  },
  "zh-Hans": {
    "local_name": "中文（简体）",
    "translations": {
      "en": "Chinese (Simplified)",
      "zh-Hans": "中文（简体）"
    }
  }
}
```

- `local_name`：该语言自有的名称
- `translations.en` / `translations.zh-Hans`：在对应 UI 语言下的显示名

### 4.2 `languages.py` 的职责

`scripts/generate/languages.py` 读取 `languages.json`，生成两样东西：

**① Rust 文件** — `crates/engine/src/languages.rs`

```rust
pub fn all_languages() -> Vec<LanguageInfo> { ... }
pub fn app_languages() -> Vec<LanguageInfo> { ... }
```

`app_languages()` 仅返回 UI 支持的语言（`en`, `zh-Hans`），用于设置界面。

**② 更新 i18n JSON** — 自动写入 `common.language` 节点：

```json
{
  "common": {
    "language": {
      "en": "English",
      "zh_Hans": "中文（简体）",
      "ja": "Japanese",
      ...
    }
  }
}
```

> `en` → `en`，`zh-Hans` → `zh_Hans`（JSON key 名中短横线替换为下划线）

### 4.3 数据流向

```
languages.json
    │
    ├─→ languages.py ──→ crates/engine/src/languages.rs
    │                     （Rust 侧：翻译引擎可用语言列表）
    │
    └─→ languages.py ──→ en.i18n.json (common.language.*)
                         zh-Hans.i18n.json (common.language.*)
                              │
                              ├─→ slang build_runner ──→ strings.g.dart（Flutter UI）
                              │
                              └─→ macos_i18n.py ──→ Localizable.strings（macOS UI）
                                                    LocaleKeys.swift（macOS UI）
```

## 5. 代码生成流程

### 5.1 完整流程

```bash
# 一键生成所有 i18n + Rust + FFI 绑定
python scripts/codegen.py
```

`codegen.py` 依次执行：

1. `scripts/generate/runtime_bindings.py`
2. `scripts/generate/languages.py`
3. `scripts/generate/macos_i18n.py`
4. `scripts/format.py`（格式化所有生成/修改的文件）

### 5.2 增量更新

仅修改了 `*.i18n.json` 后：

```bash
# 1) 更新 macOS 侧
python scripts/generate/macos_i18n.py

# 2) 更新 Flutter 侧
cd apps/desktop
dart run slang
```

仅修改了 `languages.json` 后：

```bash
python scripts/generate/languages.py
```

> `dart run slang` 将由 `slang_build_runner` 根据 JSON 源文件重新生成 Dart 侧代码。
> 也可以使用 `dart run build_runner build --delete-conflicting-outputs`，但 `dart run slang` 更快速。

## 6. 如何添加新字符串

### 6.1 步骤

**① 在 JSON 源文件中添加 key**

编辑 `{en,zh-Hans}.i18n.json`，按功能模块放置：

```json
{
  "settings": {
    "general": {
      "row": {
        "autoCopyDetectedText": "Auto Copy Detected Text"
      }
    }
  }
}
```

两个语言文件都要添加对应的翻译。

**② 重新生成**

```bash
# 生成 Flutter 侧
cd apps/desktop && dart run slang

# 生成 macOS 侧
cd /path/to/repo/root && python scripts/generate/macos_i18n.py
```

**③ 在代码中使用**

```dart
// Dart
t.settings.general.row.autoCopyDetectedText
```

```swift
// Swift
LocaleKeys.settings.general.row.autoCopyDetectedText.tr()
```

### 6.2 带参数的字符串

JSON 源文件中使用 `{}` 作为参数占位符：

```json
{
  "settings": {
    "providers": {
        "delete_dialog": {
          "title": "Delete Provider",
          "message": "Are you sure you want to delete {}?",
          "confirm": "Delete",
          "cancel": "Cancel"
        }
    }
  }
}
```

使用时传入参数：

```dart
t.settings.providers.deleteDialog.message("Google Translate")
// → "Are you sure you want to delete Google Translate?"
```

```swift
LocaleKeys.settings.providers.deleteDialog.message.tr("Google Translate")
// → "Are you sure you want to delete Google Translate?"
```

## 7. 如何添加新语言

### 7.1 UI 语言（Flutter + macOS）

目前仅支持 `en` 和 `zh-Hans`。若要新增（例如 `ja`）：

1. 创建 `apps/desktop/lib/src/i18n/ja.i18n.json`，包含所有 key 的日文翻译
2. 在 `scripts/generate/languages.py` 的 `APP_DISPLAY_LANGUAGES` 元组中添加 `"ja"`
3. 重新运行 `python scripts/codegen.py`
4. 在 Flutter 端的语言选择 UI 中添加 `ja` 选项
5. 在 macOS 侧 `AppLocale.normalize()` 中添加 `ja` 的分支

### 7.2 翻译目标语言（引擎支持的语言）

编辑 `scripts/generate/data/languages.json`，按格式添加新条目：

```json
"xx": {
  "local_name": "...",
  "translations": {
    "en": "...",
    "zh-Hans": "..."
  }
}
```

然后运行 `python scripts/generate/languages.py`。脚本会自动：
- 更新 `crates/engine/src/languages.rs`
- 更新 `en.i18n.json` 和 `zh-Hans.i18n.json` 的 `common.language.*`

## 8. 命名规范

### 8.1 关键原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **只用 `snake_case`** | JSON key 名全小写 + 下划线分隔：`auto_copy_detected_text` |
| 2 | **按 UI 组织，不按翻译者** | key 层级反映界面结构，而不是翻译管理员的思维 |
| 3 | **leaf key 就是最终字符串** | 不做字符串拼接，不要只写一半让代码补后半句 |
| 4 | **只有一个事实来源** | 同一个含义只在一个位置定义，不要在不同模块重复定义相同的语义 |
| 5 | **让翻译人员能看懂上下文** | key 路径本身就要说明这个字符串用在哪 |

### 8.2 Key 层级结构

```
{domain}.{component}.{category}.{descriptor}
              ↑          ↑           ↑
          功能模块     UI 元素类型  具体标识
```

#### Domain（顶级域）

| domain | 对应功能 | 示例 key |
|--------|---------|---------|
| `common` | 通用文案和数据标签 | `common.ui.button.ok` |
| `settings` | 设置窗口 | `settings.general.title` |
| `mini_translator` | 迷你翻译窗口 | `mini_translator.input.hint` |
| `app` | 应用全局文案（托盘、关于等） | `app.tray.context_menu.quit` |

#### Category（UI 元素类型）

| category | 适用场景 | 示例 |
|----------|---------|------|
| `ui` | UI 基础组件文案 | `common.ui.button.ok` |
| `title` | 页面/区块标题 | `settings.shortcuts.title` |
| `section` | 分区标题（Table 中的 Section header） | `settings.general.section.permissions` |
| `row` | 设置行标签（Table Row 的左侧文本） | `settings.general.row.launch_at_login` |
| `option` | 选择项（Picker/Segment 中的值） | `settings.general.option.none` |
| `button` | 按钮文字 | `mini_translator.button.clear` |
| `xxx_dialog` | 弹窗内容（`{action}_dialog` 作为一级 key，内含 `title`、`message`、`confirm`、`cancel`） | `settings.providers.delete_dialog.title` |
| `alert` | 错误/警告弹窗 | `settings.providers.alert.error` |
| `feedback` | 操作反馈（Toast/SnackBar） | `common.ui.feedback.copied` |
| `message` | 信息提示文字 | `mini_translator.message.please_enter_word_or_text` |
| `tooltip` | 鼠标悬停提示 | `mini_translator.toolbar.tooltip.translation_mode` |
| `hint` | 输入框提示 | `mini_translator.input.hint` |
| `intro` | 功能介绍文案（长文本） | `settings.providers.intro.body` |
| `capability` | 服务能力标签 | `settings.providers.capability.translation` |
| `description` | 能力详细说明 | `settings.providers.description.dictionary` |
| `context_menu` | 右键菜单 | `app.tray.context_menu.quit` |
| `empty` | 空状态提示 | `settings.layout.empty.title` |
| `editor` | 编辑/表单页面中的 label、placeholder、tooltip | `settings.providers.editor.row.id` |
| `tray` | 系统托盘 | `app.tray.context_menu` |
| `language` | 语言名称列表（自动生成） | `common.language.en` |
| `provider` | 提供商名称列表 | `common.provider.google` |
| `theme_mode` | 主题模式标签 | `common.theme_mode.dark` |
| `translation_mode` | 翻译模式标签 | `common.translation_mode.auto` |
| `word_pronunciation` | 发音标签 | `common.word_pronunciation.us` |

### 8.3 英文文案规范

#### 大小写

| 规则 | 正确 | 错误 |
|------|------|------|
| 按钮、标题、标签 → **首词大写 + 其余小写**（sentence case） | `"Launch when you log in"` | `"Launch When You Log In"` |
| 专有名词 → 保留原大小写 | `"Youdao OCR"` | `"Youdao Ocr"` |
| 完整的弹窗/提示句子 → 首字母大写结尾句号 | `"Required permissions are still missing."` | `"required permissions are still missing"` |
| 简短标签 → 不加句号 | `"Default directory service"` | `"Default directory service."` |
| 占位符文本 → 句子式 | `"Enter the word or text here"` | `"enter the word or text here"` |

#### 时态与语态

| 规则 | 正确 | 错误 |
|------|------|------|
| 按钮用**祈使句**（动词原形） | `"Add"`, `"Save"` | `"Adding"`, `"Added"` |
| 描述用**一般现在时** | `"Provides dictionary lookup"` | `"Provided dictionary lookup"` |
| 提示用**主动语态** | `"Grant Screen Recording access"` | `"Screen Recording access should be granted"` |
| 空状态用**现在时** | `"No providers configured"` | `"No providers have been configured"` |

#### 长度与简洁性

| 规则 | 正确 | 错误 |
|------|------|------|
| 按钮文案 ≤ 3 个单词 | `"Add Target"` | `"Add a new translation target"` |
| Section 标题 ≤ 4 个单词 | `"Input Settings"` | `"Settings related to text input"` |
| 设置行标签 ≤ 8 个单词 | `"Auto copy detected text"` | `"Automatically copy text that has been detected by the OCR"` |
| 长的说明文本可以完整写 | `intro.body` / `description.*` 不限 | — |
| 避免所有格（'s） | `"Provider type"` | `"Provider's type"` |

#### 一致性

同一概念在整个应用中术语必须一致：

| 应使用 | 不要用 |
|--------|-------|
| `"Detect"` | `"Identify"`, `"Recognize"`（在翻译上下文中） |
| `"Extract"` | `"Capture"`, `"Grab"`（提取屏幕文字时） |
| `"Grant"` | `"Allow"`, `"Authorize"`（授予权限时） |
| `"Service"` | `"Engine"`, `"Plugin"`（指翻译/OCR 服务提供方时） |
| `"Provider"` | `"Service Account"`, `"Integration"`（指配置的翻译账号时） |
| `"Screen Recording"` | `"Screen Capture"`（macOS 系统术语） |
| `"Accessibility"` | `"Screen Selection"`（macOS 系统术语） |

### 8.4 JSON 层级设计模式

#### 模式 A：设置页面（`settings.{page}.{type}.{name}`）

```json
{
  "settings": {
    "general": {
      "title": "General",                          // 页面标题
      "section": {                                   // Section 分组标签
        "permissions": "Permissions"
      },
      "row": {                                      // 设置项的标签
        "launch_at_login": "Launch when you log in"
      },
      "option": {                                   // 选择项的值
        "none": "None"
      },
      "button": {                                   // 页面内的按钮
        "grant": "Grant"
      }
    }
  }
}
```

#### 模式 B：弹窗（`{domain}.{component}.{action}_dialog.{field}`）

```json
{
  "settings": {
    "providers": {
      "delete_dialog": {
        "title": "Delete Provider",
        "message": "Delete \"{}\"?\nThis action cannot be undone.",
        "confirm": "Delete",
        "cancel": "Cancel"
      },
      "reset_dialog": {
        "title": "Reset Shortcuts",
        "message": "Are you sure you want to reset all shortcuts to their default values?",
        "confirm": "Reset",
        "cancel": "Cancel"
      }
    }
  }
}
```

弹窗以 `{action}_dialog` 作为 JSON key 名（Dart/Swift 侧自动转为 `{action}Dialog`），内含四个固定字段：

| 字段 | 说明 | 必填 |
|------|------|------|
| `title` | 弹窗标题 | ✅ |
| `message` | 弹窗正文 | ✅ |
| `confirm` | 确认按钮文字 | ✅ |
| `cancel` | 取消按钮文字 | ❌（仅当有取消操作时） |

> 对应的 Dart 调用：`t.settings.providers.deleteDialog.message`
> 对应的 Swift 调用：`LocaleKeys.settings.providers.deleteDialog.message.tr()`

#### 模式 C：Tooltip（`{domain}.{component}.tooltip.{name}`）

```json
{
  "mini_translator": {
    "toolbar": {
      "tooltip": {
        "translation_mode": "Current translation mode: {}",
        "extract_text_from_screen_capture": "Capture screen area and recognize text"
      }
    }
  }
}
```

工具提示（tooltip）以句子式呈现，若不含参数则不加句号。

#### 模式 D：Banner 多状态文案

```json
{
  "mini_translator": {
    "limited_banner": {
      "permission": {
        "missing_both": "Grant Screen Recording and Accessibility permissions...",
        "missing_screen_capture": "Grant Screen Recording permission...",
        "missing_accessibility": "Grant Accessibility permission..."
      },
      "action": {
        "app_settings": "App Settings",
        "recheck": "Recheck"
      },
      "feedback": {
        "enabled": "Screen text extraction is enabled.",
        "still_missing": "Required permissions are still missing."
      }
    }
  }
}
```

多状态的 banner/通知，按状态维度（如 `permission`、`feedback`、`action`、`instruction`）分组，而不是按状态编号命名。

### 8.5 参数化字符串

```
// 使用 {} 作为占位符
"mini_translator.toolbar.tooltip.translation_mode": "Current translation mode: {}"
"settings.advanced.running_at":       "Running at {url}"
"settings.providers.delete_dialog.title": "Delete \"{}\"?"
"settings.version":          "Version {} BUILD {}"
```

| 规则 | 说明 |
|------|------|
| 单个无歧义参数 → 用裸 `{}` | 简洁，Dart/Swift 端按位置传参 |
| 多个参数 → 在 value 中用注释性文字提高可读性 | `"Running at {url}"` 比 `"Running at {}"` 更清晰，但占位符本身仍然是 `{}`，`{url}` 只是标注给翻译人员看的 |
| 带引号的值 → 在 JSON 中用 `\"` 转义 | `"Delete \"{}\"?"` |
| 不要把参数作为 key 名的一部分 | ❌ `settings.providers.delete_dialog.message_google` → ✅ `settings.providers.delete_dialog.message` |

### 8.6 反模式（Anti-patterns）

| 反模式 | 示例 | 问题 | 建议 |
|--------|------|------|------|
| **字符串拼接** | `t.hello + " " + t.world` | 语序不对，翻译不可用 | 用参数 `t.greeting("world")` |
| **key 名带序号** | `message_1`, `message_2`, `instruction_3` | 增加新消息时序号混乱 | 用语义命名：`empty_message`, `error_message` |
| **key 名带语言** | `title_en`, `desc_zh_cn` | 各语言 JSON 文件之间重复信息 | JSON 文件本身就是按语言分的 |
| **过度嵌套** | `settings.general.section.ocr.option.detail.title` | 太深，Dart 侧写起来冗长 | 控制在 4-5 层以内 |
| **同义不同名** | 有的地方叫 `engine`，有的叫 `service` | 翻译人员迷惑 | 统一用 `service`（见上方术语表） |
| **HTML/XML 标签** | `"Click <b>here</b> to continue"` | Flutter 不渲染，Swift 也不渲染 | 单条完整字符串，或在代码层换行 |

### 8.7 Key 层级深度约束

| 层级 | 最大深度 | 示例 |
|------|---------|------|
| 普通 key | 4 层 | `settings.providers.editor.row.id` |
| 特殊复杂场景 | ≤ 5 层 | `mini_translator.limited_banner.permission.missing_both` |

若超过 5 层，需要重新考虑是嵌套过深还是 domain 划分不合理。

### 8.8 JSON Key ↔ 代码命名映射

| JSON 格式 | Dart `.tr()` | Swift `.tr()` |
|-----------|-------------|--------------|
| `snake_case` | `snakeCase`（slang 自动转换） | `swiftIdentifier`（macos_i18n.py 自动转换） |
| `zh_hans` | `zhHans` | `zhHans` |
| `macos_settings` | `macosSettings` | `macosSettings` |

> Dart（slang）自动将 `snake_case` 转为 `lowerCamelCase`。Swift 侧同样通过 `swift_identifier()` 函数自动转换。
> 
> **JSON 中始终写 `snake_case`，不要手动 camelCase。**

## 9. FAQ

### Q: 为什么不用 ARB 格式？

ARB 是 Flutter 官方推荐格式，但 slang 支持多种源格式（JSON、YAML、ARB），本项目历史选择了 JSON。JSON 与 macOS 侧的 Python 生成器更易集成。

### Q: Dart 和 macOS 的翻译文件会不同步吗？

理论上不会。macOS 侧的文件由 `macos_i18n.py` 从同样的 `*.i18n.json` 生成，`codegen.py` 保证两者一致。如果发现不同步，运行 `python scripts/codegen.py` 即可。

### Q: 为什么 `languages.json` 不直接放到 i18n JSON 里？

`languages.json` 除了给 UI 显示语言名称，还要给 Rust 引擎层使用（翻译哪些语言可用）。它是两个消费方（Rust + i18n）的共同数据源，单独管理更清晰，避免 Rust crate 依赖 JSON 文件解析。
